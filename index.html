Getting Started

Setup & Run

Get the app running in under 5 minutes.

1

Clone the repository

Run the git command to download the project to your machine.

git clone https://github.com/YOUR_USERNAME/uni-explorer.git
cd uni-explorer
2

Install dependencies

Installs all packages listed in package.json including Expo and React Navigation.

npm install
3

Start the development server

Expo launches a local Metro bundler and shows a QR code.

npx expo start
4

Open on your device

Press i for iOS Simulator, a for Android Emulator, or scan the QR with the Expo Go app on your phone.

Requirements: Node.js v18+, npm or yarn, and Expo Go app on your phone (for physical device testing). iOS Simulator requires macOS + Xcode.
File 01

App.js

The root entry point of the entire application.

⚡ App.js — Root Component
JS ▼

What this file does

App.js is the top-level component that React Native renders first. It wraps the entire app in SafeAreaProvider (which provides safe area insets for notches and home bars) and then renders the AppNavigator which contains all the screen routing logic.

SafeAreaProvider — must wrap the whole app so every screen can use useSafeAreaInsets()
AppNavigator — delegates all routing to the navigation file, keeping App.js clean
Expo's AppEntry.js (in package.json "main") automatically calls this component
JS 
// App.js
// Root component — sets up safe area and navigation

import React from "react";
import { SafeAreaProvider } from "react-native-safe-area-context";
import AppNavigator from "./src/navigation/AppNavigator";

export default function App() {
  return (
    <SafeAreaProvider>
      <AppNavigator />
    </SafeAreaProvider>
  );
}

What this file does

Babel is a JavaScript transpiler that converts JSX and modern ES6+ syntax into code that runs on both Android (JavaScriptCore) and iOS (Hermes). The babel-preset-expo preset includes all the transformations needed for Expo apps — JSX, class properties, decorators, and more.

babel-preset-expo — wraps @babel/preset-react and @babel/preset-typescript with Expo-specific options

  return {
    presets: ['babel-preset-expo'],
  };
};

File 07

src/hooks/useResponsive.js

Custom React hook that detects screen size and returns responsive values.

🔁src/hooks/useResponsive.js— Hook
JS▼

How responsive design works in React Native

React Native doesn't have CSS media queries. Instead, we use useWindowDimensions() which re-renders the component whenever the window size changes (e.g., when a tablet rotates). This hook abstracts that logic into easy-to-use boolean flags and values.

isTablet — true when width ≥ 768px (iPad, large Android tablets)
numColumns — 2 on tablet (grid), 1 on phone (single column)
horizontalPadding — 32px on tablet, 16px on phone
fontScale — multiplier: 0.9× on tiny phones, 1.1× on tablets
scale(size) — utility to scale pixel values proportionally to screen width
JS 
import { Dimensions, useWindowDimensions } from "react-native";

const BREAKPOINTS = {
  sm: 375,   // small phones (SE, Galaxy A)
  md: 428,   // large phones (Pro Max)
  lg: 768,   // tablets
  xl: 1024,  // large tablets / desktop
};

export function useResponsive() {
  const { width, height } = useWindowDimensions();
  // Re-runs automatically on rotation / resize

  const isTablet = width >= BREAKPOINTS.lg;

  return {
    width, height,
    isSmall:           width < BREAKPOINTS.sm,
    isTablet,
    numColumns:        isTablet ? 2 : 1,
    horizontalPadding: isTablet ? 32 : 16,
    fontScale:         width < BREAKPOINTS.sm ? 0.9
                       : isTablet ? 1.1 : 1,
  };
}

// Scale a pixel value proportionally to screen width
const BASE_WIDTH = 390;
export function scale(size) {
  const { width } = Dimensions.get("window");
  return Math.round((size * width) / BASE_WIDTH);
}
File 08

src/navigation/AppNavigator.js

Sets up React Navigation's native stack — the push/pop screen transition system.

🧭src/navigation/AppNavigator.js— Navigation
JS▼

Navigation architecture

React Navigation uses a stack metaphor: screens are pushed on top of each other. Going back pops the top screen off. NavigationContainer holds the navigation state tree. createNativeStackNavigator uses the platform's native navigation (UINavigationController on iOS, Fragment on Android) for performance.

headerShown: false — hides the built-in header; each screen has its own custom header
animation: "slide_from_right" — standard push animation (slide new screen in from right)
contentStyle — sets the background color under screens to prevent white flash
Screens receive navigation and route props automatically from the stack
JS 
import { NavigationContainer } from "@react-navigation/native";
import { createNativeStackNavigator } from "@react-navigation/native-stack";
import HomeScreen from "../screens/HomeScreen";
import DetailsScreen from "../screens/DetailsScreen";

const Stack = createNativeStackNavigator();

export default function AppNavigator() {
  return (
    <NavigationContainer>             // holds nav state
      <Stack.Navigator
        initialRouteName="Home"        // first screen
        screenOptions={{
          headerShown: false,          // custom headers
          contentStyle: {
            backgroundColor: Colors.background
          },
          animation: "slide_from_right"
        }}
      >
        <Stack.Screen name="Home"    component={HomeScreen}    />
        <Stack.Screen name="Details" component={DetailsScreen} />
      </Stack.Navigator>
    </NavigationContainer>
  );
}

src/screens/HomeScreen.js

The main screen — renders the searchable list of all universities.

📋src/screens/HomeScreen.js— Screen
JS▼

Key concepts in this screen

FlatList — used instead of ScrollView because it virtualizes the list (only renders visible items), which is efficient for long lists. It takes data, keyExtractor, and renderItem props.
useMemo — the filter runs only when searchQuery or universities changes, not on every render. This prevents unnecessary computation.
ListHeaderComponent — the header (title, stats, search bar) scrolls with the list instead of being stuck above it.
ListEmptyComponent — shown automatically by FlatList when data is an empty array.
navigation.navigate("Details", { university: item }) — passes the full university object to DetailsScreen via route params.
keyboardDismissMode="on-drag" — hides keyboard when user starts scrolling the list.
JS 
export default function HomeScreen({ navigation }) {
  const [searchQuery, setSearchQuery] = useState("");
  const { horizontalPadding, fontScale } = useResponsive();

  // Filter only when searchQuery changes (performance)
  const filteredUniversities = useMemo(() => {
    const q = searchQuery.toLowerCase().trim();
    if (!q) return universities;
    return universities.filter(u =>
      u.university.toLowerCase().includes(q) ||
      u.country.toLowerCase().includes(q) ||
      u.city.toLowerCase().includes(q)
    );
  }, [searchQuery]);

  const handleCardPress = (item) => {
    // Navigate and pass full university object as route param
    navigation.navigate("Details", { university: item });
  };

  return (
    <SafeAreaView style={styles.safeArea}>
      <FlatList
        data={filteredUniversities}
        keyExtractor={(item) => String(item.id)}
        renderItem={({ item }) =>
          <UniversityCard item={item}
            onPress={() => handleCardPress(item)} />
        }
        ListHeaderComponent={ListHeader}   // scrolls with list
        ListEmptyComponent={ListEmpty}    // shown when no results
        keyboardDismissMode="on-drag"
        showsVerticalScrollIndicator={false}
      />
    </SafeAreaView>
  );
}

src/screens/DetailsScreen.js

Full university info view with fade-in animation and all detail sections.

🔍src/screens/DetailsScreen.js— Screen
JS▼

Key concepts in this screen

route.params.university — receives the university object passed from HomeScreen via navigation.
Animated.Value — React Native's animation driver. fadeAnim goes from 0→1 (opacity), slideAnim from 24→0 (translateY).
Animated.parallel — runs both animations simultaneously so the screen fades in while sliding up.
useNativeDriver: true — runs animations on the native UI thread for 60fps performance, no JS bridge overhead.
navigation.goBack() — pops this screen off the stack, returning to HomeScreen.
accentColor + "25" — appends hex alpha to the accent color for a tinted badge background (25 = ~15% opacity).
JS 
export default function DetailsScreen({ route, navigation }) {
  const { university } = route.params;  // data from HomeScreen

  // Animation values
  const fadeAnim  = useRef(new Animated.Value(0)).current;
  const slideAnim = useRef(new Animated.Value(24)).current;

  useEffect(() => {
    Animated.parallel([
      Animated.timing(fadeAnim, {
        toValue: 1, duration: 350, useNativeDriver: true
      }),
      Animated.spring(slideAnim, {
        toValue: 0, speed: 18, bounciness: 6, useNativeDriver: true
      }),
    ]).start();
  }, []);

  return (
    <Animated.View style={{
      opacity: fadeAnim,
      transform: [{ translateY: slideAnim }]
    }}>
      // ... hero, about, stats, programs, website
    </Animated.View>
  );
}

src/components/UniversityCard.js

The most complex component — interactive card with spring press animation.

🃏src/components/UniversityCard.js— Component
JS▼

Spring press animation explained

When the user presses the card, scaleAnim shrinks to 0.97 with a fast spring. When they release, it bounces back to 1.0. This is wrapped in Animated.View with transform: [{ scale: scaleAnim }]. The TouchableOpacity handles the press detection. useNativeDriver: true runs the animation on the GPU thread for buttery 60fps.

Props: item (university object) and onPress (callback from HomeScreen)
Left accent bar — 4px wide colored bar using item.accentColor (unique to each university)
numberOfLines={2} on description — truncates with ellipsis if too long
accessibilityLabel — screen reader reads "University of Toronto, Canada. Tap to view details."
JS 
export default function UniversityCard({ item, onPress }) {
  const scaleAnim = useRef(new Animated.Value(1)).current;

  const handlePressIn = () =>
    Animated.spring(scaleAnim, {
      toValue: 0.97, useNativeDriver: true, speed: 50
    }).start();

  const handlePressOut = () =>
    Animated.spring(scaleAnim, {
      toValue: 1, useNativeDriver: true, bounciness: 6
    }).start();

  return (
    <Animated.View style={{ transform: [{ scale: scaleAnim }] }}>
      <TouchableOpacity
        onPress={onPress}
        onPressIn={handlePressIn}
        onPressOut={handlePressOut}
        accessibilityLabel={`${item.university}, ${item.country}. Tap to view details.`}
      >
        <View style={styles.card}>
          // Colored accent bar on the left
          <View style={[styles.accentBar, { backgroundColor: item.accentColor }]} />
          // ... emoji, name, location, description, ranking tag
        </View>
      </TouchableOpacity>
    </Animated.View>
  );
}

src/components/SearchBar.js

Controlled text input — value and onChange live in the parent (HomeScreen).

🔍src/components/SearchBar.js— Component
JS▼

Controlled vs uncontrolled inputs

This is a controlled component — its value is driven by parent state (searchQuery in HomeScreen). Every keystroke triggers onChangeText, which updates state, which re-renders and passes the new value back. This pattern lets the parent filter the list in real time.

clearButtonMode="while-editing" — shows iOS clear (×) button when text is present
returnKeyType="search" — shows "Search" button on keyboard instead of "Return"
accessibilityRole="search" — tells screen readers this is a search field
JS 
// Props: value, onChangeText, placeholder
export default function SearchBar({ value, onChangeText, placeholder }) {
  return (
    <View style={styles.container}>
      <Text style={styles.icon}>🔍</Text>
      <TextInput
        value={value}                          // controlled
        onChangeText={onChangeText}            // parent setState
        placeholder={placeholder}
        placeholderTextColor={Colors.textMuted}
        returnKeyType="search"
        clearButtonMode="while-editing"
        accessibilityRole="search"
      />
    </View>
  );
}
